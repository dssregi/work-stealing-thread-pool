Detailed Description

Work-stealing thread pool for parallel task execution.

The pool maintains a deque of queues (one per worker thread). Tasks are submitted to a random queue. Worker threads execute work from their own queue in LIFO order (improving cache locality), and steal from peers' queues in FIFO order when idle.

The pool uses C++20 features:

  • std::jthread for automatic thread management and joining.
  • std::stop_token for cooperative cancellation.

@thread_safety Thread pool operations are safe for concurrent task submission. Shutdown is coordinated via stop_source_ and condition variables in the underlying ThreadSafeDeque.
