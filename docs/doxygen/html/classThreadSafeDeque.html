Detailed Description

template<class T>
class ThreadSafeDeque< T >

Thread-safe work-stealing deque template.

Template Parameters
TType of the objects stored in the deque. Objects are stored as std::unique_ptr<T> internally so T must be MoveConstructible.
  • Owner threads should push and pop from the back (LIFO) to benefit from cache locality.
  • Stealing threads should call try_steal which pops from the front (FIFO) to obtain older tasks.
  • Blocking behavior is provided through push (blocks when full) and wait_and_pop (blocks until not empty or closed). try_pop and try_steal are non-blocking.

@thread_safety The class is safe for concurrent use: multiple threads may call stealing methods while a single owner thread performs owner operations. Internal synchronization is implemented with std::mutex and std::condition_variable.
