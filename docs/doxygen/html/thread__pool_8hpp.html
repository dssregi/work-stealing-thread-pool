Detailed Description

C++20 work-stealing thread pool implementation.

This header implements a thread pool using work-stealing queue semantics with C++20 structured concurrency (std::jthread and std::stop_token). Each worker thread has its own deque and performs work-stealing from randomly selected peer queues when its local queue is empty.

  • Worker threads are created at construction time based on hardware concurrency.
  • Tasks are submitted to randomly selected queues to achieve load distribution.
  • Each thread preferentially executes from its own queue (LIFO), then steals from peers (FIFO) to improve cache locality and work distribution.
  • Graceful shutdown is triggered when the destructor is called, with all pending tasks executed before thread join.
